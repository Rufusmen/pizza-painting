The Goal

Cover more cells with paint than opponent at the end of 250 turns or paint all opponent pawns.

  Rules

The board

The board is a grid, where the top-left corner is the cell (0,0). The map is randomly generated at the start of each game.

A board cell can be either:

  • empty (.) : it's not painted by any player.
  • wall (X) : cannot be painted and pawns cannot move on wall cell.
  • color (1 or 2): cell painted by a player.

Pawns

Each pawn is represented by a square grid of size 3x3.

Pawns cannot intersect.

Pawns can either MOVE or SHOOT paint.

Cells on pawns can also be painted.

Pawn is under control of a player that has more paint on it.

Players gets information only about owned pawns. All pawns are visible on board as colored cells.

Movement

Player can give MOVE command with direction to pawn.

direction is one of:

  • up (1)
  • down (2)
  • right (3)
  • left (4)

Shooting paint

Each pawn has tank with paint. Tanks have unlimited size. Initial amount of paint is same for each pawn and is generated by game engine.

Player can give SHOOT command with direction and length.

This will cause coloring strip of size 1 x min(paint, length, maxShootDist) counting from edge of the pawn.

Strip of paint will be stopped by board ends and walls.

At the end of each turn paint is added to each pawn's tank. Number of added paint units is equal to the number of board cells in pawn's color that pawn is standing on.

  Expert Rules

The source code can be found here: github.

All MOVE actions are resolved simultaneously.

During movement some collisions might happen.

  • pawns can be out of board borders
  • pawns may collide with walls
  • there might be collision between two or more pawns

All colliding pawns are moved back to their previous positions.

Collision check and rollback procedure is launched until there are no collisions left.

SHOOT actions are resolved after pawns movement.

At the end of the turn change of the ownership is resolved before the tank refill.

If one cell will be covered by two paint strips of different color it won't change it's color at all.

  The Game Input

Initialization input

Line 1: four integers color: your color, rows:number of board rows, cols: number of board columns and maxShootDist maximal shooting distance.

Next rows lines: one string with size cols: description of board row.

Game turn input

Line 1: one int diffsCnt: number of fields that changed color during last turn

Next diffsCnt lines:

  • int x: x position of cell
  • int y: y position of cell
  • char color: new cell color

Next line: one int pawnsCnt: number of pawns under your control.

Next pawnsCnt lines: pawns description:

  • int id: id of pawn
  • int paint: number of paint units in tank
  • int x: x position of center of pawn
  • int y: y position of center of pawn
  • word(9) pawnColor: 9 chars with description of pawn cells colors. 3 for each row.
Output
pawnsCnt lines (a single line for each owned unit).
  • MOVE id direction
  • SHOOT id direction length
Constraints
Allotted response time to output is ≤ 50ms.
Allotted response time to output on the first turn is ≤ 1000ms.
rows and cols are integers from range [10,50].
Each player will start with same number of pawns from rang [1,3].
maxShootDist is an integer from range [5,10].
